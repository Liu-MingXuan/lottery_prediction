# 彩票预测系统 — 详细开发流程文档

## Context

根据 README.md 的需求，从零构建一个彩票预测系统。系统通过爬取体彩大乐透和福彩双色球的历史开奖数据，使用概率统计方法（不使用机器学习/深度学习）预测下一期各号码的出现概率，并输出一组概率最大的推荐号码。

---

## 一、项目结构

```
caipiao/
├── README.md
├── requirements.txt          # 依赖包
├── config.py                 # 全局配置（API地址、数据库路径等）
├── db.py                     # SQLite 数据库操作（建表、增删改查）
├── crawler_ssq.py            # 双色球数据爬虫
├── crawler_dlt.py            # 大乐透数据爬虫
├── analyzer.py               # 概率统计分析引擎
├── predictor.py              # 预测模块（输出推荐号码）
├── main.py                   # 主入口（爬取 → 分析 → 输出预测）
└── lottery.db                # SQLite 数据库文件（运行后生成）
```

## 三、开发步骤

### 第1步：基础设施（config.py + db.py + requirements.txt）

**requirements.txt 依赖：**
- `requests` — HTTP 请求
- `beautifulsoup4` — HTML 解析（备用）
- 无其他重型依赖

**config.py 内容：**
- 双色球 API 地址：`https://www.cwl.gov.cn/cwl_admin/front/cwlkj/search/kjxx/findDrawNotice`
- 大乐透 API 地址：`https://webapi.sporttery.cn/gateway/lottery/getHistoryPageListV1.qry`
- 数据库路径：`lottery.db`
- 请求超时、User-Agent 等配置

**db.py 内容：**
- `init_db()` — 建表（如果表不存在）
- `get_latest_issue(lottery_type)` — 查询数据库中已有的最新期号，用于增量爬取
- `insert_ssq(records)` — 批量插入双色球记录
- `insert_dlt(records)` — 批量插入大乐透记录
- `get_all_ssq()` — 查询全部双色球历史
- `get_all_dlt()` — 查询全部大乐透历史

### 第2步：爬虫模块

#### crawler_ssq.py（双色球爬虫）

**API 信息：**
- 端点：`GET https://www.cwl.gov.cn/cwl_admin/front/cwlkj/search/kjxx/findDrawNotice`
- 需要先访问 `https://www.cwl.gov.cn/ygkj/wqkjgg/ssq/` 获取 session cookie
- 请求头需带 `Referer` 和 `X-Requested-With: XMLHttpRequest`
- 参数：`name=ssq`, `issueCount`, `pageNo`, `pageSize`

**逻辑流程：**
1. 用 `requests.Session()` 先访问双色球主页，获取 cookie
2. 查询数据库最新期号 `latest_issue`
3. 调用 API 分页获取数据（从最新到最旧）
4. 过滤掉 `issue <= latest_issue` 的记录（只保留新数据）
5. 解析 JSON → 提取期号、日期、红球、蓝球
6. 存入数据库

#### crawler_dlt.py（大乐透爬虫）

**API 信息：**
- 端点：`GET https://webapi.sporttery.cn/gateway/lottery/getHistoryPageListV1.qry`
- 无需认证，直接调用
- 参数：`gameNo=85`, `pageSize`, `pageNo`

**逻辑流程：**
1. 查询数据库最新期号 `latest_issue`
2. 调用 API 分页获取数据
3. 过滤掉 `issue <= latest_issue` 的记录
4. 解析 `lotteryDrawResult` 字段（空格分隔，前5个为前区，后2个为后区）
5. 存入数据库

### 第3步：概率统计分析引擎（analyzer.py）

使用纯概率统计方法，不使用任何 ML/DL。包含以下分析方法：

#### 3.1 基础频率分析
- 统计每个号码在历史中出现的**总次数**和**出现概率**（出现次数 / 总期数）
- 分别统计前区/红区、后区/蓝区的概率

#### 3.2 近期趋势分析（加权频率）
- 对最近 N 期（如最近 50 期、100 期）的数据单独统计频率
- 给近期数据更高的权重（近期 > 远期），体现"热号/冷号"趋势
- 权重方案：使用线性衰减，最近一期权重为 N，最远一期权重为 1

#### 3.3 遗漏值分析
- 计算每个号码的**当前遗漏值**（距离上次出现的期数）
- 计算**平均遗漏值**（历史平均间隔期数）
- 遗漏值越大 → 该号码近期未出现 → 回补概率理论上越高

#### 3.4 综合概率计算
对每个号码，综合以上三个维度计算最终概率：

```
综合概率 = w1 × 基础频率 + w2 × 加权频率 + w3 × 遗漏回补概率
```

- 默认权重：`w1=0.3, w2=0.4, w3=0.3`（可根据效果调整）
- 遗漏回补概率 = 当前遗漏 / 平均遗漏（归一化后）
- 最终对每个号码的概率做归一化处理，使同一区域所有号码概率之和为 1

**输出格式（以双色球为例）：**
```json
{
    "01": { "红区概率": 0.032, "蓝区概率": 0.065 },
    "02": { "红区概率": 0.028, "蓝区概率": 0.058 },
    ...
    "33": { "红区概率": 0.025, "蓝区概率": null }
}
```

### 第4步：预测模块（predictor.py）

#### 4.1 号码推荐逻辑

**双色球（6红+1蓝）：**
1. 根据红区综合概率，从高到低取前 6 个号码作为推荐红球
2. 根据蓝区综合概率，取概率最高的 1 个号码作为推荐蓝球
3. 对推荐组合计算综合中奖概率（各号码概率的乘积）

**大乐透（5前区+2后区）：**
1. 根据前区综合概率，从高到低取前 5 个号码作为推荐前区
2. 根据后区综合概率，从高到低取前 2 个号码作为推荐后区
3. 对推荐组合计算综合中奖概率

#### 4.2 输出格式

```json
{
    "双色球": {
        "红区": ["24", "27", "09", "16", "32", "06"],
        "蓝区": ["07"],
        "红区各号概率": {
            "24": 0.035, "27": 0.034, "09": 0.033,
            "16": 0.032, "32": 0.031, "06": 0.030
        },
        "蓝区各号概率": {
            "07": 0.068
        },
        "综合概率": 0.0000052
    },
    "大乐透": {
        "前区": ["25", "29", "24", "27", "34"],
        "后区": ["06", "02"],
        "前区各号概率": {
            "25": 0.036, "29": 0.034, "24": 0.033,
            "27": 0.032, "34": 0.031
        },
        "后区各号概率": {
            "06": 0.092, "02": 0.088
        },
        "综合概率": 0.0000041
    }
}
```

### 第5步：主入口（main.py）

```
流程：
1. 初始化数据库（建表）
2. 爬取双色球最新数据（增量）
3. 爬取大乐透最新数据（增量）
4. 对双色球进行概率分析 → 输出预测
5. 对大乐透进行概率分析 → 输出预测
6. 格式化输出结果到控制台
```

## 四、关键实现细节

### 4.1 增量爬取策略
- 每次启动时查询数据库最新期号
- 只爬取比该期号更新的记录
- 如果数据库为空，则从第1期开始全量爬取（分页处理，注意请求间隔避免被封）

### 4.2 双色球反爬处理
- 使用 `requests.Session()` 维持会话
- 先访问主页获取 cookie
- 设置合理的请求头（User-Agent、Referer）
- 请求间隔 1-2 秒

### 4.3 概率计算注意事项
- 所有概率值归一化，确保同区域号码概率之和为 1
- 综合概率仅作为参考指标，不构成实际中奖保证
- 权重参数可通过 config.py 调整

## 五、验证方案

1. **爬虫验证**：首次运行后检查数据库记录数（双色球应有 ~3000+ 条，大乐透 ~2800+ 条）
2. **增量验证**：再次运行，确认不会重复爬取已有记录
3. **概率验证**：检查所有号码概率之和是否约为 1.0
4. **输出验证**：确认输出的号码数量正确（双色球6红1蓝，大乐透5前2后），且号码在合法范围内
5. **端到端验证**：运行 `python main.py` 完整走通流程
