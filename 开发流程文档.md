# 彩票预测系统 — 详细开发流程文档

## Context

根据 README.md 的需求，从零构建一个彩票预测系统。系统通过爬取体彩大乐透和福彩双色球的历史开奖数据，使用概率统计方法（不使用机器学习/深度学习）预测下一期各号码的出现概率，并输出多组概率最大的推荐号码。

### 3.3 crawler/ — 爬虫模块

**数据源：500.com**（福彩官网和体彩官网均有 WAF 拦截，无法直接使用）

| 文件 | 说明 |
|---|---|
| `ssq.py` | 爬取双色球历史数据，解析 HTML 表格 |
| `dlt.py` | 爬取大乐透历史数据，解析 HTML 表格 |

**爬取逻辑：**
1. 查询数据库最新期号
2. 如果有数据，只爬取比最新期号更新的记录（增量）
3. 如果数据库为空，全量爬取（双色球约 3440 条，大乐透约 2859 条）
4. 解析 HTML 表格，提取期号、号码，存入 SQLite

### 3.4 analysis/analyzer.py — 概率分析引擎

**三种分析方法：**

#### 基础频率分析
统计每个号码在给定记录中的出现次数 / 总期数。

#### 近期加权频率分析
使用线性衰减权重（近期 > 远期），最近一期权重最大。

#### 遗漏值分析
计算每个号码的当前遗漏值（距上次出现的期数）和平均遗漏值，遗漏越大理论上回补概率越高。

**综合概率计算：**
```
综合概率 = 0.3 × 基础频率 + 0.4 × 加权频率 + 0.3 × 遗漏回补概率
```
最终归一化，使同区域所有号码概率之和为 1。

**多阶段概率展示（multi_period_analysis）：**
输出每个号码在不同历史跨度（最近10期、50期、100期、所有历史、预测概率）的概率表格，格式如下：
```
号码 | 最近10期 | 最近50期 | 最近100期 | 所有历史 | 预测
01   |  0.2000  |  0.2200  |   0.2100  |  0.1933  | 0.0427
02   |  0.4000  |  0.2600  |   0.2700  |  0.1890  | 0.0378
...
```

### 3.5 analysis/predictor.py — 预测模块

#### 子组合频率惩罚（核心算法）

**原理：** 历史中出现过的号码组合，再次出现的概率应该下降。组合越大惩罚越重。

**步骤：**
1. 预计算历史中所有 2~N 元子组合的出现频率（`build_combination_freq`）
2. 对候选号码集合，计算其所有子组合在历史中的频率加权和（`calc_penalty`）
3. 惩罚权重：2元组=1, 3元组=2, 4元组=3, 5元组=4, 6元组=5

```
惩罚分 = Σ weight(k) × freq(sub_combination) / total_draws
         k=2..N
```

#### 贪心选号（greedy_select）

从概率最高的 20 个号码中逐个选取，综合考虑：
- **个体概率**：该号码的综合预测概率
- **历史子组合惩罚**：当前已选号码 + 候选号码构成的子组合在历史中出现的频率
- **组间重叠惩罚**：与前面已选组的号码重叠数量 × 0.05（引导不同组选不同组合）

每步选 `score = 个体概率 - 历史惩罚 - 组间重叠惩罚` 最大的号码。

#### 多组预测

每组独立贪心选最优号码（允许跨组重复），通过组间重叠惩罚让各组组合尽量不同。蓝球/后区按概率排名轮流分配给各组。

**输出示例：**
```
【双色球】
  第1组: 05 11 20 26 29 33 + 09  (综合概率: 7.41e-09)
  第2组: 04 05 15 20 23 28 + 01  (综合概率: 1.32e-09)
  第3组: 03 13 21 26 27 30 + 15  (综合概率: 1.89e-10)
  ...

【大乐透】
  第1组: 01 14 15 23 31 + 03 04  (综合概率: 5.84e-08)
  第2组: 01 03 05 10 19 + 03 04  (综合概率: 1.14e-08)
  ...
```

## 四、运行流程（main.py）

```
1. 初始化数据库（建表）
2. 爬取双色球最新数据（增量）
3. 爬取大乐透最新数据（增量）
4. 多阶段概率分析 → 打印红区/蓝区概率表格
5. 多阶段概率分析 → 打印前区/后区概率表格
6. 生成5组双色球预测（贪心 + 子组合惩罚）
7. 生成5组大乐透预测（贪心 + 子组合惩罚）
8. 输出预测摘要 + 详细概率数据
```

运行命令：
```bash
source .venv/bin/activate
python main.py
```

## 五、验证方案

1. **爬虫验证**：首次运行后检查数据库记录数（双色球 ~3440 条，大乐透 ~2859 条）
2. **增量验证**：再次运行，确认新增 0 条记录
3. **概率验证**：检查各阶段概率表格中"合计"行是否约为对应号码个数（红区≈6，蓝区≈1）
4. **预测验证**：确认每组号码数量正确（双色球6红1蓝，大乐透5前2后），号码在合法范围内
5. **子组合惩罚验证**：检查5组之间是否有合理的差异（不是完全相同，也不是强制不重复）
6. **端到端验证**：`python main.py` 完整走通，无报错
